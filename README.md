

1. Реализация Тестового задания

1.1. Работает из локали, домен не был выдан

1.2. Отправка писем на почту: Из коробки НЕ АКТИВНА (Не считаю возможным оставлять в проекте возможность отправлять от моего имени коммерческую рассылку). Тестовая отправка писем в консоль отработана.

1.3. Юнит тесты НЕ РЕАЛИЗОВАНЫ в полной и необходимой мере.
Выборочно реализованы несколько тестов.
Причина: для проекта вне рамок тестового задания необходимо добавление бизнес логина и, возможно, прокидка глобальных хуков + наложение контролирующего сервера для отчетности, логов и тд. Эндпоинты, вероянтно, будут сильно изменены. Самое главное: в ТЗ не описан механизм как добавления новой виртуальной машины, так и её отключения. Требует, скорее всего, отдельного админского интерфейса и изменений почти во всем проекте, вплоть до фронтенда пользователя.

1.4. Фронтенд. Так как не требовалось, оставлен максимально минималистичным в пределах разумного.

1.5. Возможно, для нормальной работы потребуется настройка .env ! (см. пункт 5)



2. Запуск
   
2.1. Контейнеры относящиеся к сайту - docker-compose up -d --build

2.2. Точка входа: http://localhost:5173/register

2.3. Запуск десктопа: стандартный .exe. Исходник прилагается в проекте.
Исходник: /desctop_get_proxy/desctop_app.py
Сам файл: /desctop_get_proxy/dist/desktop_app.exe

2.4. NOTE: в зависимости от системы, настроек пользователя и версии ОС результат может отличаться. Тестировалось на ПК. Возможно, потребуется запуск от имени администратора, иначе может быть невозможно изменить файлы реестра пользователя.

3. Swagger работает после запуска, http://127.0.0.1:8000/api/docs/. Не реализован полностью, причина см. 1.3

4. Основная структура:
   
/config - основные настройки

/core - бэкэнд + тесты бэкэнда

/frontend - Vue фронтенд


6. Настройка .env
   
Поля с предложенными дефолтными значениями:

DEBUG=True 

SECRET_KEY=django-insecure-lalala-change-later

DATABASE_URL=postgres://admin:secret@db:5432/get_proxy_db

REDIS_URL=redis://redis:6379/0

EMAIL_HOST_USER=

EMAIL_HOST_PASSWORD=

DEFAULT_FROM_EMAIL=


8. Особенности реализации

Backend: Django REST Framework + Django Channels (WebSockets для real-time статусов).

Frontend: Vue 3.

Desktop: CustomTkinter + интеграция с системным реестром Windows для управления прокси.

Очереди: Celery + Redis для асинхронных задач.
